Story 2.3 — View and Edit Financial Information
As a user, I want to view and edit my financial information after onboarding so I can correct mistakes and keep my plan up to date.
Acceptance Criteria
Display
A dedicated “Financial Profile” or “Your Inputs” section exists
Displays:
Age, retirement age, lifespan
Current assets (grouped by type)
Monthly contributions
Income streams
Retirement spending target
Values shown match the data used in the latest projection
Editing
User can edit any field via:
Inline edit OR
“Edit” modal / form
Fields are validated (same rules as onboarding)
Save & Recompute
On save:
Inputs are persisted
Projection automatically re-runs
Results screen updates
Recompute completes in ≤2 seconds
UX Safeguards
Unsaved changes are clearly indicated
User can cancel edits without affecting the current plan
Clear distinction between:
Editing reality (financial profile)
Exploring scenarios (what-if analysis)
